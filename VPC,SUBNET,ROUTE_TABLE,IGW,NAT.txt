1.CREATE VPC
    ASSIGN CIDR BLOCK
2.CREATE TWO SUBNET
	1.PUBLIC 
	2.PRIVATE
3.CREATE INTERNET GATEWAY
4.ATTACH VPC TO INTERNET GATEWAY
5.CREATE TWO SEPRATE ROUTE TABLES
	1.PUBLIC RT
	2.PRIVATE RT
EDIT ROUTES ADD IGW FOR PUBLIC RT
6.CREATE TWO EC2 INSTANCE 
	1. SELECT VPC WHICH YOU HAVE CREATED.
	2. LAUNCH ONE EC2 INSTANCE WITH PUBLIC-SUBNET
7.AFTER LAUNCHING THE EC2 INSTANCE, CONNECT THE EC2 INSTANCE AND IN UBUNTU LOGIN USE PWD TO GET THE WORKING DIRECTORY.
8.ON SYSTEM, OPEN THE TERMINAL WHERE THE DOWNLOAD KEY IS AND OPEN COMMAND PROMPT/TERMINAL AND RUN THIS COMMAND TO TRANSFER THIS KEY TO THE EC2 INSTANCE USING SCP(SECURE COPY PROTOCOL) COMMAND  
	scp -i /path/key-pair-name.pem /path/my-file.txt ec2-user@instance-public-dns-name:path/
	eg. scp -i key_mumbai.pem key_mumbai.pem ubuntu@10.0.0.11:/home/ubuntu
9.RUN THE EC2 PUBLIC INSTANCE USING TERMINAL 
	chmod 600 key_mumbai.pem
	ssh -i key_mumbai.pem ubuntu@<private-ec2-ip>
10.TO GIVE INTERNET GATEWAY TO PRIVATE EC2, WE NEED NAT GATEWAY
11.CREATE NAT GATEWAY
12.EDIT ROUTE TABLE 
	IN PRIVATE RT, EDIT ROUTE ADD NAT GATEWAY
13. FROM EC2 PING WWW.GOOGLE.COM
